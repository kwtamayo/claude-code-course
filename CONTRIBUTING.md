# Contributing to Claude Code Course

Thank you for your interest in contributing! This project aims to make software development accessible to everyone.

## How to Contribute

### Reporting Issues
- Check if the issue already exists
- Provide clear steps to reproduce
- Include your macOS version and any error messages
- Screenshots are helpful!

### Suggesting Enhancements
- Explain the problem you're trying to solve
- Describe your proposed solution
- Consider how it fits with the weekend-intensive timeline

### Improving Content
- Fix typos or unclear explanations
- Add troubleshooting tips based on your experience
- Suggest better examples or analogies
- Improve code samples

### Adding Features
- Discuss major changes in an issue first
- Keep changes focused and atomic
- Update documentation
- Test on a fresh macOS installation if possible

## Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/claude-code-course.git
   cd claude-code-course
   ```

3. Install dependencies:
   ```bash
   npm install
   ```

4. Run the development server:
   ```bash
   npm run dev
   ```

5. Make your changes

6. Test your changes

7. Commit and push:
   ```bash
   git add .
   git commit -m "Description of your changes"
   git push origin main
   ```

8. Open a Pull Request

## Course Content Guidelines

### Writing Lessons
- Use clear, simple language (8th-grade reading level)
- Include practical examples students can relate to
- Add validation tasks for hands-on learning
- Provide helpful error messages and hints
- Include troubleshooting sections
- Keep time estimates accurate

### Lesson Structure
Every lesson should have:
- Clear learning objectives
- Step-by-step instructions
- Validation tasks
- Troubleshooting section
- Key concepts summary
- Links to next/previous lessons

### Code Examples
- Keep examples minimal and focused
- Add comments explaining what's happening
- Use realistic variable names
- Follow JavaScript/React best practices
- Test all code before committing

## Style Guidelines

### Markdown
- Use headers properly (H1 for title, H2 for sections, etc.)
- Include code blocks with proper syntax highlighting
- Use emoji sparingly and meaningfully
- Break up long paragraphs

### Code Style
- Follow the existing code style
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Validation Tasks

### Writing Validation
- Make tasks specific and measurable
- Provide clear success criteria
- Include helpful hints
- Handle both success and failure cases gracefully

### Testing Validation
- Test with both correct and incorrect inputs
- Verify error messages are helpful
- Ensure validation doesn't block progress unfairly

## Pull Request Process

1. Update the README.md if needed
2. Update time estimates if lesson duration changes
3. Add yourself to CONTRIBUTORS.md
4. Follow the PR template
5. Wait for review and address feedback

## Code of Conduct

### Our Standards
- Be welcoming and inclusive
- Respect different viewpoints and experiences
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior
- Harassment or discrimination
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
- Conduct that could be deemed unprofessional

## Questions?

Feel free to:
- Open an issue for discussion
- Start a conversation in GitHub Discussions
- Reach out to the maintainers
